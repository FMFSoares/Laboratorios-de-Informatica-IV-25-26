2. Execute the rewritten prompt
3. After changes are applied, append to this file: the rewritten prompt + a brief explanation of what was done

For minor prompts (questions, explanations, status checks, clarifications): treat normally, nothing gets written here.

### [2026-05-08] Mechanic single active timer constraint

**Prompt:**
Implement a backend constraint that prevents a mechanic from having more than one active timer running simultaneously across different service orders. When `POST /ordens-servico/{id}/tempos/iniciar` is called, the system must check whether the mechanic assigned to that OS already has a running timer on any other OS. If a conflict is found, reject the request with a `409 MECANICO_TIMER_CONFLICT` error that includes the conflicting OS's `id` and `numero` in the response body, so the frontend can later present a confirmation dialog asking the user whether to stop the old timer and start the new one, or continue working on the existing OS.

**What was done:**
- Added a conflict check in `iniciar_tempo` in `ordem_servico_service.py`: after verifying the current OS has no active timer, scans all other OS assigned to the same mechanic for a running `inicio_tempo_atual`
- Returns `409` with `code: MECANICO_TIMER_CONFLICT` and both `os_conflito_id` and `os_conflito_numero` so the frontend has enough context to build the confirmation dialog
- Updated the router doc for `POST /{os_id}/tempos/iniciar` to document the new error code
- Frontend dialog left for when OS views are built
