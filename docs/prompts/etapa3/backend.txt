   Documentar:
   - POST /api/v1/ordens-servico

   - GET /api/v1/ordens-servico?loja_id={id}&estado={estado}
   - PATCH /api/v1/ordens-servico/{id}/estado
   - POST /api/v1/ordens-servico/{id}/tempos/iniciar
   - POST /api/v1/ordens-servico/{id}/tempos/parar

   Definir estes estados:
   - PENDENTE
   - EM_DIAGNOSTICO
   - AGUARDA_APROVACAO
   - EM_REPARACAO
   - AGUARDA_PECAS
   - CONCLUIDA
   - FATURADA
   - CANCELADA

   - PENDENTE → EM_DIAGNOSTICO
   - EM_DIAGNOSTICO → AGUARDA_APROVACAO

   - EM_REPARACAO → AGUARDA_PECAS
   - AGUARDA_PECAS → EM_REPARACAO
   - EM_REPARACAO → CONCLUIDA

   - preco_servico é o preço comercial/tabelado do serviço prestado.

   - Não pode ser emitida fatura para OS que não esteja CONCLUIDA.
   - Não pode existir mais do que uma fatura para a mesma OS.

   A response da fatura deve discriminar:

   - serviço prestado
   - preco_servico
   - peças aplicadas
